# ETL-процесс сбора вакансий с сайта hh.ru

## Описание проекта

ETL-процесс для инкрементального сбора данных о вакансиях с платформы [hh.ru](https://hh.ru).  
Скрипт загружает только новые вакансии с момента последнего успешного запуска, сохраняет результаты в БД PostgreSQL и в файловую систему в колоночном формате Parquet со сжатием.  
Процесс устойчив к сетевым сбоям из-за механизма ретраев, ведёт логирование и отправляет уведомление о результатах в Telegram.

## Функционал

### ETL
- **Инкрементальная загрузка** – использует максимальную `request_dttm` из целевой таблицы для определения даты начала поиска
- **HTTP-запросы к hh.ru** – формирует URL с параметрами фильтрации (регион, текст поиска, поле поиска, сортировка по дате публикации)
- **Парсинг HTML** – извлекает JSON-данные
- **Трансформация данных** – преобразует временные метки в UTC, формирует структурированные записи
- **Сохранение в PostgreSQL** – через `polars.write_database()`
- **Сохранение в Parquet** – каждая страница пишется в отдельный файл в папке `data/` с сжатием

### Логирование
- Логи пишутся одновременно в файл `hh_search.log` и в консоль.
- Каждое сообщение лога содержит **уникальный идентификатор запуска** (`etl_id`) для трассировки.
- Логируются начало и конец работы, параметры поиска, количество обработанных страниц, найденных вакансий, ошибки.

### Обработка ошибок и ретраи
- Реализован **самодельный декоратор `retry`**
- При сбое HTTP-запроса выполняется до **5 повторных попыток** с интервалом **2 секунды**
- Каждая неудачная попытка и финальная ошибка логируются

### Вынос конфигурации
- Все настройки вынесены в отдельный yaml файл `conf/config.yaml`
- Чтение делаем с помощью библиотеки `PyYAML`.

### Генерация ключа `etl_id`
- При каждом запуске генерируется uuid.
- `etl_id` добавляется в лог, в колонку таблицы `hh_search` и в имя файлов Parquet

### Уведомления в Телеграм
- По окончании работы (успех или ошибка) отправляется сообщение в Телеграм
- Если Телеграм недоступен, процесс не прерывается, просто в логах это пишем, т.к. телеграм работает через раз

## Настройка

### 1. Подключение к БД
Надо создать внутри БД табличу hh_search с такими полями:
```sql
create schema if not exists etl;

create table if not exists etl.hh_search (
    request_dttm      timestamp,
    etl_id            uuid,
    vacancy_id        bigint,
    vacancy_title     text,
    company_id        bigint,
    company_title     text,
    company_visible_name text,
    publication_time  timestamp,
    last_change_time  timestamp,
    creation_time     timestamp,
    is_adv            text,
    snippet           text,
    responses_count   integer,
    total_responses_count integer
);
```

### 3. Установка зависимостей
```
pip install -r requirements.txt
```
### 4. Запуск скрипта
```
python etl_script.py
```

Скрипт:

1. Определит дату последней загрузки из БД

2. Выполнит поиск вакансий на hh.ru опубликованных после этой даты

3. Сохранит данные в таблицу etl.hh_search и в файлы Parquet в папке data

4. Отправит отчет в Telegram

## Логирование
Все события записываются в файл hh_search.log и дублируются в консоль. Пример записи:
```
2026-04-10 02:31:29,809 - hh_search - INFO - [3025029a-003c-4dbf-9ac5-f4bf77ab5834] Searching vacancies from 01.01.2026 00:00:00 (etl_id=3025029a-003c-4dbf-9ac5-f4bf77ab5834)
2026-04-10 02:31:30,760 - hh_search - INFO - [3025029a-003c-4dbf-9ac5-f4bf77ab5834] Processing page 0
2026-04-10 02:31:33,837 - hh_search - INFO - [3025029a-003c-4dbf-9ac5-f4bf77ab5834] Processing page 1
```
## Уведомления в Telegram
При успешном завершении в Telegram придёт сообщение вида:
```
<b>Результаты поиска вакансий (etl_id=3025029a-003c-4dbf-9ac5-f4bf77ab5834)</b>
Начало поиска: 2026-04-09 23:31:29.492086+00:00
Обработано страниц: 49
Новых вакансий: 961
Время выполнения: 52 секунды
```
Если Telegram недоступен или, процесс продолжит работу – в лог будет записано предупреждение.
